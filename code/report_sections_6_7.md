**第6节：泛化测试结果分析**

本章节对训练好的 PointNet 模型进行了鲁棒性测试，主要考察模型在受到外界干扰（高斯噪声）和数据缺失（随机丢点）时的分类精度表现。

**6.1 噪声鲁棒性测试**
实验通过向测试集点云数据加入不同强度的零均值高斯噪声（σ = 0, 0.01, 0.02, 0.05）来模拟现实扫描中的传感器误差。测试结果如下表所示：

| 干扰类型 (基准) | 参数 | Overall Accuracy |
| :--- | :--- | :--- |
| 无干扰 (Baseline) | - | 88.54% |
| 高斯噪声 | σ=0.01 | 87.78% |
| 高斯噪声 | σ=0.02 | 86.06% |
| 高斯噪声 | σ=0.05 | 66.91% |

**分析：**
- 当噪声强度较小时（σ=0.01），模型精度仅下降约 0.76%，表现出极强的抗噪能力。
- 随着噪声增加到 σ=0.05，精度大幅下降至 66.91%。这说明 PointNet 虽然对轻微噪声具有鲁棒性，但在高噪声环境下，由于其完全依赖全局最大池化（Max Pooling）提取特征，局部细节的严重扭曲会导致全局特征失效。

**6.2 随机丢点鲁棒性测试**
实验通过对测试集点云随机丢弃一定比例的点（0%, 10%, 30%, 50%）来模拟数据采集不完整的情况。

| 干扰类型 | 丢点比例 | Overall Accuracy |
| :--- | :--- | :--- |
| 无丢点 | 0% | 88.54% |
| 随机丢点 | 10% | 88.30% |
| 随机丢点 | 30% | 88.10% |
| 随机丢点 | 50% | 87.86% |

**分析：**
- PointNet 在丢弃高达 50% 的点时，精度依然保持在 87.8% 以上，仅下降了不到 1%。
- 这充分验证了 PointNet 的核心优势：通过对称函数（Max Pooling）提取全局特征。只要物体的主要骨架点（Critical Points）未被完全丢弃，模型就能保持极高的分类稳定性。

**第7节：失败案例分析与改进建议**

**7.1 典型失败案例展示**
通过对 279 个错误分类样本的统计，我们发现模型在处理形状高度相似或具有包含关系的类别时容易产生混淆。以下是前三名最常见的“混淆对”：

1. **Glass_box vs. TV_stand (玻璃箱 vs. 电视柜):** 两者都具有长方体框架结构，在点云密度较低时，模型无法区分玻璃的透明特性或电视柜的内部隔板细节。
2. **Plant vs. Flower_pot (植物 vs. 花盆):** 这是一对典型的包含关系。花盆中通常包含植物，而植物也常生长在盆中。点云特征在整体轮廓上高度重合，导致模型难以界定分类边界。
3. **Dresser vs. Night_stand (梳妆柜 vs. 床头柜):** 两者的本质区别在于尺寸比例和抽屉布局，但在归一化后的点云空间中，这种比例差异变得不明显。

**7.2 失败原因分析**
1. **局部几何特征缺失:** PointNet 独立处理每个点，缺乏对局部领域（Local Neighborhood）的建模。对于像“花盆”和“植物”这种需要通过局部纹理或精细连接关系来区分的类别，全局特征力有不逮。
2. **尺度不变性的负面影响:** 为了实现平移和缩放不变性，点云被统一归一化。这导致原本可以靠体积区分的物体（如大型梳妆柜与小型床头柜）变得不可区分。
3. **关键点遮挡/缺失:** 在某些视角下，物体的特征部位（如椅子的扶手或桌子的抽屉）点云分布稀疏，模型仅靠残缺的全局轮廓进行推断。

**7.3 改进建议**
1. **引入局部特征聚合:** 建议采用 PointNet++，通过多层级的分组（Grouping）和采样（Sampling）来捕捉局部几何结构。
2. **增加法向量辅助:** 利用点云的法向量信息（Normals）来增强对表面平滑度和曲率变化的感知，这能有效提升“花盆”与“植物”的区分度。
3. **细粒度数据增强:** 针对易混淆类别，在训练时加入非均匀缩放和局部随机抖动，强制模型学习更本质的几何特征而非简单的比例。
4. **损失函数加权:** 针对混淆严重的类别对，在训练时引入更高的惩罚权重，引导模型关注细微的判别性区域。
